一、物理层与基本参数
•	接口：UART（异步串行）
•	波特率：4800 bps
•	数据位：8
•	校验位：无（None）
•	停止位：1 
•	字节序：MODBUS RTU 为大端模式（先发送高字节）
•	从机地址：默认 0x01
________________________________________
二、MODBUS RTU 帧格式
地址码 (1B)	功能码 (1B)	数据区 (nB)	CRC16 校验 (2B)
•	CRC16：低字节在前，高字节在后的“小端”模式（MODBUS 标准）
________________________________________
三、支持的功能码
仅使用 03 (读保持寄存器) 和 06 (写单个寄存器)，符合工业电子秤常用子集。
功能码	说明
0x03	读取 HX711 数据或配置值
0x06	写单个寄存器（如去皮、校准）
________________________________________
四、寄存器地址定义
寄存器地址 (HEX)	名称	访问类型	说明
0x0000	ADC 原始值	只读	4 字节 (实际 HX711 为 24 位，对齐到 16–31 位可自行扩展)
0x0002	去皮值	读写	4 字节，用于手动设置或读取当前去皮量
0x0004	校准系数	读写	4 字节，单位：克/LSB（用户可缩放为浮点，单片机内用定点数乘除处理）
			
0x0006	实时重量（克）	只读	计算后 = (原始值 - 去皮值) × 校准系数
0x0007	控制与状态	读写/位	Bit0: 1=自动去皮；Bit1: 1=稳定状态；Bit2: 1=允许连续输出
说明：若 HX711 是 24 位数据，建议将 0x0000 扩展为 2 个连续寄存器（32 位），地址使用 0x0000(高字)+0x0001(低字)。
为简单示例，下文中 0x0000 仅返回 (ADC >> 8) 的 16 位值。
________________________________________
五、命令示例
1. 读取实时重量（从机地址 0x01）
请求：
原：01 03 00 03 00 01 CRC_LO CRC_HI
改：01 03 00 04 00 01 CRC_LO CRC_HI
响应（重量 = 1024 克）：
01 03 02 04 00 CRC_LO CRC_HI
（04 00 表示 0x0400 = 1024 十进制）
2. 设置去皮值（设置去皮值 = 200）
请求：
原：01 06 00 01 00 C8 CRC_LO CRC_HI
改：01 06 00 02 00 C8 CRC_LO CRC_HI
响应：相同帧，表示写入成功
3. 读取 ADC 原始值
请求：
01 03 00 00 00 01 CRC_LO CRC_HI
响应：
01 03 02 1A 2F CRC_LO CRC_HI（0x1A2F = 6703）

六、异常处理
异常响应：
地址码 | 0x83 (针对0x03时错误) | 异常码 | CRC
异常码	含义
0x01	非法功能码
0x02	非法寄存器地址
0x03	数据值超出范围
________________________________________
七、特殊注意事项（针对 HX711 + 4800bps）
1.	读取间隔：
HX711 转换速度典型 10~80 SPS，MODBUS 轮询建议 ≥ 100ms/次。
2.	CRC16 算法：
使用 CRC-16-MODBUS (多项式 0x8005，初始 0xFFFF，不异或输出)。
3.	数据刷新机制：
建议单片机内部以 ≥10Hz 更新 ADC 值，MODBUS 查询时直接返回最新缓存，不要在中断中等待 HX711。
4.	去皮逻辑：
收到“自动去皮”(控制寄存器 Bit0 置1) 时，MCU 应将当前重量值写入去皮寄存器。
5.	稳定判断：
可增加平滑滤波，当连续几次波动 < 阈值时，状态寄存器 Bit1 置 1。
________________________________________
八、协议实现建议（给下位机设计者）
•	接收超时 ≥ 3.5 字符时间（4800bps ≈ 3.5 * (10bit/4800) ≈ 7.3ms），使用串口空闲中断或定时器。
•	只响应地址匹配的帧，不回复正常帧以外的错误帧。
•	支持广播地址 0x00（可选，这里不要求）。
•	写寄存器立即生效并写入 EEPROM？若需要则增加延时，否则掉电丢失。

稳定状态控制位怎么使用？
稳定状态”是指称重传感器的输出值（重量）已经处于一个很小的波动范围内，不再剧烈变化。换句话说：
当用户把物体静止放在秤上后，系统判断“读数基本不变了”，这个时刻就称为“稳定状态”。
在电子秤或数据采集模块中，这是一个非常实用的功能 —— 上位机或用户不需要一直等待数据“慢慢停住”，而是可以直接查询“稳定位”是否为 1，确认当前重量可信，然后快速读取重量值。
________________________________________
1. 为什么需要“稳定状态”位？
在没有稳定位的情况下：
•	上位机读重量 → 可能是抖动中的值，比如 502g、498g、505g 来回跳。
•	如果要记录稳定后的重量，上位机需要自己实时监测数据变化，逻辑较复杂。
有了稳定位：
•	单片机自己进行滤波+抖动检测。
•	当数据变化范围连续多次小于“稳定阈值”（例如 ±1g）时，置位“稳定标志”。
•	上位机读到稳定位=1 → 知道当前重量是可信的。
•	如果刚放物体或取下物体，稳定位=0。
________________________________________
2. 稳定状态的典型判定逻辑（下位机实现）
伪代码示例：
c
static int prev_weight = 0;
static int stable_counter = 0;

void update_stable_flag(int current_weight) {
    int delta = abs(current_weight - prev_weight);

    if (delta < STABLE_THRESHOLD) {   // 例如小于 1g
        stable_counter++;
        if (stable_counter >= STABLE_COUNT_LIMIT) {
            status_register.bit_stable = 1;   // 稳定位置1
        }
    } else {
        stable_counter = 0;
        status_register.bit_stable = 0;       // 不稳定
    }

    prev_weight = current_weight;
}
•	STABLE_THRESHOLD：稳定阈值（如 0.5g、1g）
•	STABLE_COUNT_LIMIT：连续多少次才确认稳定（防瞬时稳定假象，比如连续 5 次）
每次置稳定位后，当重量再次变化超过阈值，马上清除稳定位。
________________________________________
3. MODBUS 中如何使用“稳定状态”
根据前面给出的寄存器定义，地址 0x0005 控制与状态寄存器 的 Bit1 表示稳定状态。
•	Bit1 = 1：当前重量已稳定
•	Bit1 = 0：重量未稳定（可能在加载、卸载、或晃动）
使用步骤示例：
1.	上位机读取 0x0004 寄存器
→ 得到状态字节，判断 bit1 = 1 吗？
2.	如果 bit1 = 1
→ 立即去读 0x0003（实时重量）
→ 记录重量值
3.	如果 bit1 = 0
→ 上位机可以等待一小段时间（如 50~100ms）再重复步骤 1
这样省去上位机自己做稳定判断的逻辑。
________________________________________
4. 典型场景解释
场景	稳定位
空秤静止	1
刚放上 500g 砝码（前 0.5 秒）	0
放好砝码 2 秒后	1
手轻压秤面并保持静止	1
正在用手拍秤面	0
取下物体过程中	0
取下后再静止	1
这只是一种 软件辅助标志，不要求外部参考信号，全靠单片机对 HX711 数据的实时滤波与判断。
________________________________________
5. 在协议文档中的表达建议
如果你要写正式的协议文档，可以这样描述“稳定状态”：
稳定状态：表示 HX711 当前输出的 ADC 值（或换算后的重量）在时间上已连续满足“变化量小于预设稳定阈值”的条件，且保持超过指定的判定时长。该标志由下位机通过数字滤波与窗口比较自动更新，上位机可将其作为读取有效重量值的依据；当重量再次出现超过阈值的变化时，稳定状态自动清零。
